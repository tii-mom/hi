HI is not a crypto exchange.

HI is not a trading dashboard.

HI is not a SaaS admin panel.

HI is:

# A Living AI Financial Operating System.

## What This Means For The Frontend

The frontend must make continuous AI operation visible. The user should see agents reasoning, memory being recalled, consensus forming, market conditions being interpreted, risk being enforced, and autonomous activity continuing across desktop, mobile, and Telegram.

Charts, wallets, portfolio views, agent marketplaces, and notifications are interface surfaces inside a larger operating system. They are not the product identity by themselves.

## Purpose

Provide the top-level frontend interpretation of HI so every implementation document, component, and route points toward the same product identity.

## UX Goals

The frontend should communicate continuous AI operation, autonomous coordination, and financial intelligence before generic trading actions.

## Technical Implementation Strategy

Keep the protocol repository as the source of truth and use this repo for implementation-oriented frontend guidance. Convert protocol-facing ideas into UI view models, components, motion semantics, and responsive behavior.

## Component Structure

The system is composed from terminal, agents, portfolio, skills, risk, consensus, mobile, Telegram, design, and shell components. Each surface should expose its role in the operating system.

## State Management Strategy

Frontend state should present AI activity, user preferences, localization, Telegram context, navigation, and view-model state. Protocol and runtime state remain upstream and enter the app through adapters.

## Animation Behavior

Animation should express living AI operation through pulse, flow, scan, wave, glow, fracture, and ripple semantics.

## Mobile Behavior

Mobile is a primary experience with bottom navigation, one-hand interaction, and responsive terminal stacking.

## Telegram Behavior

Telegram is a native operating surface with WebView constraints, deep links, inline AI notifications, wallet handoff, and safe-area handling.

## Performance Considerations

The operating system feeling must be efficient. Use route splitting, isolated renders, GPU-friendly motion, compact mobile loading paths, and reduced motion support.

## Source Of Truth Boundary

The Human-Intelligence repository remains the source of truth for protocol, runtime, governance, AI civilization, security, identity, token, and execution architecture.

The `hi` repository owns frontend implementation: UI system, UX system, animations, components, responsive behavior, localization, mobile experience, and Telegram WebApp behavior.

## Frontend North Star

The product should feel like Apple Human Interface Guidelines, Bloomberg Terminal system docs, OpenAI Operator architecture, and AI-native financial infrastructure combined into one coherent surface.

The system message the frontend must communicate is:

> "My AI financial civilization is alive and operating continuously."
